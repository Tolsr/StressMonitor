# Contributing to Stress Monitor

感谢你对 Stress Monitor 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 bug，请在 GitHub Issues 中创建一个 issue，并包含以下信息：

1. **问题描述** - 简要描述你遇到的问题
2. **复现步骤** - 详细列出如何复现这个问题
3. **期望行为** - 描述你期望的正确行为
4. **实际行为** - 描述实际发生的情况
5. **环境信息** - macOS 版本、应用版本等

### 提交功能建议

我们欢迎新功能建议！请在创建 issue 时：

1. 说明这个功能解决什么问题
2. 描述你期望的功能表现
3. 如果可能，提供设计草图或示例

### 提交代码

1. **Fork** 本仓库
2. **创建分支**: `git checkout -b feature/your-feature-name`
3. **编写代码**: 确保代码风格与项目一致
4. **测试**: 确保你的更改不会破坏现有功能
5. **提交**: `git commit -m "Add: your feature description"`
6. **推送**: `git push origin feature/your-feature-name`
7. **创建 PR**: 在 GitHub 上创建 Pull Request

## 代码规范

### Swift 代码风格

- 使用 4 空格缩进
- 遵循 [Swift API Design Guidelines](https://swift.org/documentation/api-design-guidelines/)
- 使用有意义的变量和函数名
- 添加必要的注释说明复杂逻辑

### 提交信息格式

```
类型: 简短描述

详细说明（可选）

相关 issue（可选）
```

**类型包括：**
- `Add`: 新功能
- `Fix`: Bug 修复
- `Update`: 更新现有功能
- `Remove`: 删除功能或代码
- `Refactor`: 代码重构
- `Docs`: 文档更新

## 开发环境设置

1. 安装 Xcode 14.0 或更高版本
2. 克隆仓库: `git clone https://github.com/YOUR_USERNAME/StressMonitor.git`
3. 打开项目: `open StressMonitor.xcodeproj`
4. 按 ⌘R 运行项目

## 行为准则

请保持友好和尊重。我们致力于为所有人提供一个友好、安全和包容的社区环境。

## 许可证

通过贡献代码，你同意你的贡献将按照 MIT 许可证发布。
